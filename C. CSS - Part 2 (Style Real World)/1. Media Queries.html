CSS Basics — Media Queries & Breakpoints

This lesson covers how to make your website look good on every screen size — from an iPhone to a 4K TV. You'll learn media queries, breakpoints, and the viewport meta tag.

Why Does This Matter?

People visit websites on all kinds of screens:

Phone
Tablet
Laptop
Desktop / TV

If your site only looks good on a laptop, you're ignoring over half your visitors. Media queries let you write CSS rules that only apply at certain screen sizes.

Step 1: The Viewport Meta Tag

Before media queries work properly, you need this line in your <head>:

<meta name="viewport"
  content="width=device-width, initial-scale=1.0">

What it does: tells the phone to use its actual screen width instead of pretending to be a desktop. Without this, your phone zooms out to fit a 1200px layout into a 390px screen — making everything tiny.

This tag goes in every HTML file you create. Always. No exceptions.

Step 2: Your First Media Query

A media query is CSS wrapped in an @media rule. The CSS inside only applies when the condition is true.

/* This applies to ALL screens */
h1 {
  font-size: 36px;
}

/* This ONLY applies when the screen is 768px or narrower */
@media (max-width: 768px) {
  h1 {
    font-size: 24px;
  }
}

Read it like English: "When the screen is at most 768px wide, make h1 smaller."

Here's a live example — resize your browser window to see the change:

Resize your browser — I change size!

↑ On a wide screen this is large with a gray background. On a narrow screen (≤768px) it shrinks and turns green.

Step 3: max-width vs min-width

There are two ways to write media queries. They do opposite things:

max-width

"When the screen is this size or smaller"

Desktop-first approach: start big, then override for small screens.

min-width

"When the screen is this size or larger"

Mobile-first approach: start small, then add styles for big screens.

/* Desktop-first: start with desktop styles,
   then override for smaller screens */
.sidebar {
  width: 250px;       /* desktop: sidebar is 250px */
}
@media (max-width: 768px) {
  .sidebar {
    width: 100%;      /* tablet and below: full width */
  }
}


/* Mobile-first: start with mobile styles,
   then add styles for larger screens */
.sidebar {
  width: 100%;        /* mobile: full width */
}
@media (min-width: 769px) {
  .sidebar {
    width: 250px;     /* tablet and above: sidebar is 250px */
  }
}

Both approaches work. Most modern frameworks use mobile-first (min-width), but desktop-first (max-width) is easier to learn. Pick one and be consistent.

Step 4: What's a Breakpoint?

A breakpoint is the screen width where your layout changes. It's the number inside max-width or min-width.

@media (max-width: 768px) { ... }
                    ↑
              this is the breakpoint

You choose breakpoints based on where your design starts to look bad — not based on specific devices. But it helps to know what sizes real devices are.

Step 5: Common Breakpoints

Here are the standard breakpoints most developers use. You don't need all of them — pick 2 or 3 that make sense for your design.

Category Breakpoint Targets
Small phones 480px iPhone SE, older Android phones
Phones 768px All phones in portrait mode
Tablets 1024px iPad, Android tablets, phones in landscape
Laptops 1280px Small laptops, standard desktops
Desktops 1440px Large desktop monitors
Large screens 1920px Full HD monitors, TVs 
/* A simple 3-breakpoint system covers most cases */

/* Phones (up to 768px) */
@media (max-width: 768px) { ... }

/* Tablets (up to 1024px) */
@media (max-width: 1024px) { ... }

/* Big screens (1440px and up) */
@media (min-width: 1440px) { ... }

Tip: you almost never need more than 3 breakpoints. Start with just 768px — that covers the phone-to-desktop jump, which is the most important one.

Wait — CSS Pixels vs Physical Pixels?

This confuses everyone at first. Here's the simple version:

Physical pixels

The actual tiny dots on the screen. An iPhone 15 has 2556 × 1179 of them.

CSS pixels

What CSS sees. An iPhone 15 reports 393 × 852 CSS pixels. This is what media queries use.

Modern screens pack multiple physical pixels into each CSS pixel. This makes things look sharp (Retina / HiDPI), but it means the numbers in your @media rules are smaller than you'd expect.

You only need to think about CSS pixels. The device handles the rest.

Practical Example: Responsive Card Grid

Let's build a card grid that adapts from 1 column on phones to 3 columns on desktop.

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}

/* Tablet: 2 columns */
@media (max-width: 1024px) {
  .grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* Phone: 1 column */
@media (max-width: 600px) {
  .grid {
    grid-template-columns: 1fr;
  }
}

Practical Example: Responsive Text Sizing

Text that looks great on desktop is often too big on a phone. Scale it down at breakpoints.

Welcome to our site

This heading and paragraph change size when the screen gets narrower. Try resizing your browser.

h1 {
  font-size: 48px;
}
p {
  font-size: 18px;
}

@media (max-width: 768px) {
  h1 {
    font-size: 28px;
  }
  p {
    font-size: 15px;
  }
}

Tip: you can also use clamp() for fluid sizing without breakpoints: font-size: clamp(24px, 5vw, 48px). But breakpoints give you more control.

Practical Example: Hiding & Showing Elements

Some things should only appear on certain screen sizes. Use display: none inside a media query.

✓ I only appear on desktop (above 768px)
✓ I only appear on mobile (768px and below)

↑ Resize your browser to see which one shows.

.desktop-only {
  display: block;
}
.mobile-only {
  display: none;
}

@media (max-width: 768px) {
  .desktop-only { display: none; }
  .mobile-only  { display: block; }
}

Common uses: hide a sidebar on mobile, show a hamburger menu only on phones, or display a "download the app" banner only on mobile.

Practical Example: Responsive Spacing

Spacing that works on a big screen feels too spread out on a phone. Tighten it up with media queries.

.section {
  padding: 80px 60px;     /* lots of room on desktop */
}

@media (max-width: 1024px) {
  .section {
    padding: 48px 32px;   /* less on tablet */
  }
}

@media (max-width: 768px) {
  .section {
    padding: 32px 16px;   /* tight on phone */
  }
}

A good rule of thumb: cut your padding roughly in half at each breakpoint.

Practical Example: Two Columns → One Column

The most common layout shift: side-by-side on desktop, stacked on mobile.

About Our Company

We build tools that help creators ship faster. On desktop, this text sits beside the image. On mobile, they stack vertically.

↑ Resize below 600px to see them stack.

.two-column {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 32px;
}

@media (max-width: 768px) {
  .two-column {
    grid-template-columns: 1fr;
  }
}

Order Matters!

When using max-width, always put larger breakpoints first.

/* ✅ Correct order (largest to smallest) */
@media (max-width: 1024px) { ... }   /* tablets */
@media (max-width: 768px)  { ... }   /* phones */
@media (max-width: 480px)  { ... }   /* small phones */


/* ❌ Wrong order */
@media (max-width: 480px)  { ... }
@media (max-width: 768px)  { ... }   /* this overrides the one above! */

CSS reads top-to-bottom. If two rules both apply, the last one wins. So with max-width, go from largest to smallest. With min-width, go from smallest to largest.

Bonus: Skip Breakpoints Entirely

For card grids, there's a CSS Grid trick that makes them responsive without any media queries:

.grid {
  display: grid;
  grid-template-columns:
    repeat(auto-fit, minmax(250px, 1fr));
  gap: 16px;
}

This single line says: "make as many columns as will fit, each at least 250px wide." The browser handles the rest. This is the most useful CSS Grid trick you'll learn.

Practical Example: max-width Makes Layouts Responsive Automatically

One of the easiest ways to make layouts responsive is by combining:

  • Flexible layouts (Flexbox or Grid)
  • width: 100%
  • max-width

This prevents elements from getting too wide on large screens while still allowing them to shrink naturally on small screens.

Think of max-width as a safety limit:

width: 100%;
max-width: 400px;

This means:

  • Grow up to 400px
  • But shrink smaller if needed

Without max-width, layouts often stretch too much on large monitors and look awkward.

Example 1: Flexbox Cards

Flex Card 1
Flex Card 2
Flex Card 3

↑ Resize your browser. The cards wrap automatically and never grow too wide.

        .flex-container {
        display: flex;
        flex-wrap: wrap;
        justify-content: center;
        gap: 16px;
        }

        .card {
        flex: 1 1 250px;
        max-width: 320px;
        }

What's happening?

  • flex-wrap: wrap allows cards to move onto new rows
  • flex: 1 1 250px says:
    • grow if space exists
    • shrink if needed
    • prefer being around 250px wide
  • max-width: 320px prevents giant stretched cards on large monitors

This creates responsive behavior with very little code.

Example 2: Grid + max-width Container

Grid Card A
Grid Card B
Grid Card C
Grid Card D

↑ The grid automatically changes columns while the whole layout stays centered and readable.

.wrapper {
  max-width: 900px;
  margin: 0 auto;
}

.grid {
  display: grid;
  grid-template-columns:
    repeat(auto-fit, minmax(220px, 1fr));
  gap: 16px;
}

Why this works so well:

  • max-width: 900px stops content from becoming too stretched
  • margin: 0 auto centers the layout
  • auto-fit + minmax() automatically adjusts the number of columns
  • The layout becomes responsive without many media queries

This is how many modern websites stay readable on:

  • Phones
  • Tablets
  • Laptops
  • Ultrawide monitors
  • 4K displays

Modern responsive CSS is often less about tons of breakpoints and more about:

  • Flexible layouts
  • Wrapping content
  • Using sensible max-width values

Starter Template

Copy this as a starting point for any responsive page. It includes the viewport tag, a 3-breakpoint system, and the most common responsive patterns.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport"
    content="width=device-width, initial-scale=1.0">
  <title>My Responsive Page</title>
  <style>

    /* ===== Base styles (all screens) ===== */
    * { margin: 0; padding: 0; box-sizing: border-box; }
    body { font-family: Arial, sans-serif; }

    .container {
      max-width: 1200px;
      margin: 0 auto;
      padding: 0 40px;
    }

    h1 { font-size: 48px; }
    p  { font-size: 18px; }

    .grid {
      display: grid;
      grid-template-columns: repeat(3, 1fr);
      gap: 24px;
    }

    /* ===== Tablet (1024px and below) ===== */
    @media (max-width: 1024px) {
      .container { padding: 0 24px; }
      h1 { font-size: 36px; }
      .grid { grid-template-columns: repeat(2, 1fr); }
    }

    /* ===== Phone (768px and below) ===== */
    @media (max-width: 768px) {
      .container { padding: 0 16px; }
      h1 { font-size: 28px; }
      p  { font-size: 16px; }
      .grid { grid-template-columns: 1fr; }
    }

  </style>
</head>
<body>

  <div class="container">
    <h1>Hello World</h1>
    <p>This page is responsive.</p>

    <div class="grid">
      <div>Card 1</div>
      <div>Card 2</div>
      <div>Card 3</div>
    </div>
  </div>

</body>
</html>

Quick Reference

Here's everything you learned in this lesson:

  • <meta name="viewport"> — required for responsive to work on phones
  • @media (max-width: ___) — styles apply at that width and below
  • @media (min-width: ___) — styles apply at that width and above
  • Breakpoint — the pixel value where your layout changes
  • CSS pixels ≠ physical pixels — media queries use CSS pixels
  • Order matters — with max-width, go largest → smallest

The breakpoints to remember:

  • 768px — the phone/tablet cutoff (most important)
  • 1024px — the tablet/laptop cutoff
  • 1440px — for large desktop optimizations

The device ranges to remember:

  • Phones: 360–430px (portrait)
  • Tablets: 744–1024px (portrait)
  • Laptops: 1280–1728px
  • Desktops: 1920–2560px
  • 4K / TVs: 3840px

And the patterns you'll use most:

  • 3 columns → 2 columns → 1 column (card grids)
  • Side-by-side → stacked (text + image sections)
  • Shrink font sizes at each breakpoint
  • Reduce padding/spacing on smaller screens
  • Hide/show elements (sidebar, hamburger, banners)
  • auto-fit + minmax() — responsive grid with zero breakpoints
