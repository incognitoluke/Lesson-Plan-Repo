CSS Basics — Building a Navbar

This lesson shows you how to build a navigation bar from scratch — starting with plain links and ending with a real, responsive navbar you'd see on any website.

Step 1: Start With Links

A navbar is really just a group of links. Let's start with a simple list.

Home About Work Contact 
<nav>
  <a href="#">Home</a>
  <a href="#">About</a>
  <a href="#">Work</a>
  <a href="#">Contact</a>
</nav>

Right now they're just default blue links stacked inline. Not very navbar-like yet.

Step 2: Remove the Underlines

Links have underlines by default. Remove them with text-decoration: none.

Home About Work Contact 
nav a {
  text-decoration: none;
  color: #333;
}

We also changed the color from the default blue to dark gray. Cleaner already.

Step 3: Add Spacing Between Links

The links are too close together. Use margin or padding to space them out.

Home About Work Contact 
nav a {
  text-decoration: none;
  color: #333;
  margin-right: 24px;
}

Now there's breathing room. But later we'll use gap with Flexbox — it's even cleaner.

Step 4: Add a Background Color

A navbar needs to stand out from the page. Give it a background-color and padding.

Home About Work Contact 
nav {
  background-color: #2c3e50;
  padding: 16px;
}

nav a {
  color: white;
}

Dark background + white text is the most common navbar pattern. You can also do light background + dark text.

Step 5: Use Flexbox for Layout

Flexbox makes it easy to align items in a row and control spacing. Add display: flex to the navbar.

Home About Work Contact 
nav {
  display: flex;
  align-items: center;
  gap: 24px;
  background-color: #2c3e50;
  padding: 16px;
}

display: flex puts everything in a row. gap: 24px adds equal space between each link — no more individual margins needed. align-items: center vertically centers everything.

The magic is justify-content: space-between — it pushes the logo to the left edge and the links to the right edge.

Step 7: Style the Links Better

Let's make the links look more polished with font-weight, font-size, and some padding so they're easier to click.

MyBrand
Home About Work Contact 
.nav-links a {
  text-decoration: none;
  color: rgba(255, 255, 255, 0.85);
  font-size: 0.95em;
  font-weight: 500;
  padding: 8px 14px;
}

We used rgba(255, 255, 255, 0.85) instead of plain white — it's slightly transparent, which looks softer. Adding padding makes the clickable area bigger (better for phones).

Step 8: Add Hover Effects

Links should react when you hover over them. This tells the user "hey, you can click me." Use the :hover pseudo-class.

Home About Work Contact

↑ Hover over the links above to see the effect.

.nav-links a {
  text-decoration: none;
  color: rgba(255, 255, 255, 0.7);
  padding: 8px 14px;
  border-radius: 6px;
  transition: background-color 0.2s, color 0.2s;
}

.nav-links a:hover {
  background-color: rgba(255, 255, 255, 0.1);
  color: white;
}

transition makes the change smooth instead of instant. Without it, the color would snap on and off — less polished.

Step 12: Show the Active Page

Users should be able to see which page they're on. Use an .active class on the current link.

<a href="/about">About</a>

  color: white;
  background-color: rgba(255, 255, 255, 0.15);
}

You add the active class to whichever link matches the current page. On the About page, you'd move it to the About link.

Step 13: Animated Underline on Hover

A popular effect: an underline that slides in when you hover. This uses the ::after pseudo-element.

MyBrand
Home About Work Contact

↑ Hover over the links to see the underline slide in.

.nav-links a {
  position: relative;
  padding: 8px 0;
}

.nav-links a::after {
  content: '';
  position: absolute;
  bottom: 0;
  left: 0;
  width: 0;
  height: 2px;
  background-color: #e74c3c;
  transition: width 0.3s;
}

.nav-links a:hover::after {
  width: 100%;
}

Here's how it works: ::after creates an invisible element (width: 0). On hover, we change its width to 100%. The transition animates the change. No JavaScript needed.

This is the only part that needs JavaScript — and it's just one line: classList.toggle('open'). Everything else is CSS.

Step 15: Understanding @media Queries

The @media rule lets you write CSS that only applies at certain screen sizes.

/* These styles apply to ALL screen sizes */
.navbar {
  display: flex;
  justify-content: space-between;
  padding: 14px 20px;
}

/* These ONLY apply when the screen is 768px or narrower */
@media (max-width: 768px) {
  .nav-links {
    display: none;
  }
  .hamburger {
    display: flex;
  }
}

Think of it like this:

  • Above 768px (desktop/tablet) → show the links, hide the hamburger
  • Below 768px (phone) → hide the links, show the hamburger

768px is a common breakpoint. You can use any number that makes sense for your design.

Quick Reference

Here's everything you learned in this lesson:

  • display: flex — puts items in a row
  • justify-content: space-between — pushes logo left, links right
  • align-items: center — vertically centers everything
  • gap — adds space between flex items
  • text-decoration: none — removes link underlines
  • transition — makes hover effects smooth
  • :hover — styles that apply on mouse-over
  • ::after — creates a pseudo-element (for animated underlines)
  • position: sticky — makes the navbar stick when scrolling
  • z-index — controls stacking order (higher = on top)
  • @media — applies styles only at certain screen sizes
  • classList.toggle() — the one line of JS for the mobile menu

And these design principles:

  • Use rgba() for link colors — slightly transparent white looks softer than pure white
  • Add padding to links — bigger click targets are easier to use
  • Make the CTA button a contrasting color — draw the eye to the action you want
  • Use .active class — show users which page they're on
  • Pick border OR shadow for separation — not both
