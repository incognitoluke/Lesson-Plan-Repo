Loading Google Fonts <link>

Setup
Preview

Playfair Display

Loaded from Google Fonts in one <link> tag.

Code
/* 1. Go to fonts.google.com 2. Pick fonts & weights 3. Copy the <link> into <head> */ <link href="https://fonts.googleapis.com /css2?family=Playfair+Display :wght@400;700&display=swap" rel="stylesheet">
The <link> tag goes inside <head>, before your <style> block.

Font Pairing

Concept
Preview

Playfair Display

Paired with Outfit for body text. Serif headline + sans-serif body is a classic combo.

Unbounded

Paired with Newsreader italic. Geometric display + editorial serif creates tension.

Code
/* Load both fonts in one <link> tag */ <link href="https://fonts.googleapis.com /css2?family=Playfair+Display :wght@700&family=Outfit :wght@400&display=swap" rel="stylesheet"> h1 { font-family: 'Playfair Display', serif; } p { font-family: 'Outfit', sans-serif; }
Pair fonts with contrast: serif + sans-serif, heavy + light, display + body. Don't mix two similar fonts.

Letter Spacing / Tracking letter-spacing

Spacing
Preview

Tight tracking

Normal tracking (default)

Wide tracking — labels

Extra wide

Code
/* Negative = tighter, positive = looser */ .headline { letter-spacing: -0.03em; /* tight */ } .label { letter-spacing: 0.15em; /* wide */ text-transform: uppercase; } .spacious { letter-spacing: 4px; /* px works too */ }
ValueWhen To Use
-0.02em to -0.05emLarge headlines — tighten for impact
0Body text — leave at default
0.05em to 0.2emUppercase labels, small caps, captions
Use em so spacing scales with font size. Uppercase text almost always needs extra tracking.

Word Spacing word-spacing

Spacing
Preview

Tight word spacing makes text feel compressed and urgent in a paragraph.

Normal word spacing is what you get by default — the browser picks it.

Wide word spacing adds an airy, editorial feeling to each line.

Code
.tight { word-spacing: -2px; } .normal { word-spacing: normal; } .airy { word-spacing: 8px; }
Less commonly adjusted than letter-spacing, but useful for fine-tuning justified text or stylistic effects.

Kerning font-kerning

Spacing
Preview

Kerning: auto (default)

WAVE Type AV

Kerning: none

WAVE Type AV

Code
/* Kerning adjusts space between specific letter PAIRS (AV, To, Wa) */ .auto { font-kerning: auto; } /* browser decides */ .on { font-kerning: normal; } /* force on */ .off { font-kerning: none; } /* force off */
Kerning vs Tracking
font-kerningAdjusts specific pairs (AV, To) — built into the font file
letter-spacingUniform spacing between ALL letters
Most browsers kern by default. You rarely need to change this — but it's good to know it exists.

Text Transform text-transform

Transform
Preview

uppercase — great for labels

LOWERCASE — forces all letters down

capitalize each word in a title

None — text stays as written in HTML

Code
.label { text-transform: uppercase; letter-spacing: 0.1em; /* always add! */ } .lower { text-transform: lowercase; } .title { text-transform: capitalize; } .reset { text-transform: none; }
ValueEffect
uppercaseALL CAPS — labels, buttons, nav items
lowercaseall lowercase — stylistic choice
capitalizeFirst Letter Of Each Word
noneKeeps text as-is (override/reset)
Use CSS instead of typing caps in HTML — easier to change later, and screen readers handle it better.

Fluid Type clamp()

Fine Control
Preview

This headline scales with the viewport — bigger on wide screens, smaller on narrow.

↕ Resize your browser to see it change.

Code
/* clamp(minimum, preferred, maximum) */ h1 { font-size: clamp(1.2rem, 4vw, 3rem); } p { font-size: clamp(0.9rem, 1.5vw, 1.1rem); }
ArgumentMeaning
minNever shrinks below this (e.g. 1rem)
preferredScales with viewport (vw)
maxNever grows above this (e.g. 3rem)
Fluid type replaces media queries for font sizes. One line handles all screen widths.

Leading line-height

Fine Control
Preview

Leading of 1.0 — lines touch each other, which gets hard to read quickly in longer passages of body text.

Leading of 1.6 — comfortable for body copy. Enough air to let the eye track from line to line naturally.

Big headlines need tight leading like 1.1

Code
body { line-height: 1.6; /* body text */ } h1 { line-height: 1.1; /* headlines */ }
ContextGood Range
Headlines1.0 – 1.2
Body text1.5 – 1.8
Small / caption text1.4 – 1.6
A unitless number is a multiplier of the font-size. Bigger type → tighter leading. Smaller type → looser.

White Space white-space

Fine Control
Preview

normal (wraps)

This long text wraps normally at the edge of its container, breaking into multiple lines as needed.

nowrap (single line)

This long text never wraps — it stays on one line and gets cut off with an ellipsis if you combine it with overflow hidden and text-overflow.

Code

ValueEffect

nowrapCollapses whitespace, no wrapping
prePreserves whitespace and line breaks (like code)
pre-wrapPreserves whitespace, but wraps

Text Rendering text-rendering

Fine Control
Preview

optimizeSpeed

Affluent → fi fl ffi

optimizeLegibility

Affluent → fi fl ffi

Code
/* Enable ligatures & kerning */ body { text-rendering: optimizeLegibility; } /* Anti-aliasing (WebKit/macOS) */ body { -webkit-font-smoothing: antialiased; -moz-osx-font-smoothing: grayscale; }
ValueEffect
optimizeSpeedFastest — skips ligatures and kerning
optimizeLegibilityEnables ligatures (fi, fl) and kerning
geometricPrecisionPrecise scaling — best for animations
Add -webkit-font-smoothing: antialiased to the body for sharper text on Mac screens.
