# 🍕 Project: Build a Restaurant Menu Page

## Overview

Congratulations — you've finished Lessons 1 through 6! You now know how to write headings, paragraphs, links, images, lists, forms, and tables. This project puts all of it together.

**Your mission:** Build a one-page website for a made-up restaurant. You pick the name, the food, the vibe — everything. The only rule is you have to use the HTML elements listed below.

---

## What You're Building

A single `index.html` file that looks like a real restaurant's website. It should have:

1. **A restaurant name and tagline** at the top
2. **A short "About Us" paragraph** describing the restaurant
3. **An image** (a photo of food, the restaurant, anything that fits)
4. **A menu** displayed in a table with item names, descriptions, and prices
5. **A list of hours** the restaurant is open
6. **An order form** where a customer can place a takeout order
7. **A footer** with a link to the restaurant's (fake) social media

---

## Requirements Checklist

Use this list to make sure you've included everything. Check them off as you go.

### From Lesson 1 — HTML Skeleton
- [ ] Your file starts with `<!DOCTYPE html>`
- [ ] You have `<html>`, `<head>`, `<body>` tags
- [ ] You have a `<title>` in the `<head>`

### From Lesson 2 — Text
- [ ] At least one `<h1>` for the restaurant name
- [ ] At least two `<h2>` tags for section titles (like "Our Menu", "Hours", "Order Online")
- [ ] At least two `<p>` tags
- [ ] Use `<strong>` or `<em>` somewhere (bold a special dish, italicize a tagline, etc.)
- [ ] Use `<hr>` to separate sections

### From Lesson 3 — Links & Images
- [ ] At least one `<img>` with a `src` and `alt` attribute
- [ ] At least one `<a>` link (can go to a real site or just `#`)

### From Lesson 4 — Lists
- [ ] One `<ul>` (unordered list) — maybe for hours, toppings, or fun facts
- [ ] One `<ol>` (ordered list) — maybe for "Top 3 Sellers" or steps to order

### From Lesson 5 — Forms & Inputs
- [ ] A `<form>` tag wrapping the order section
- [ ] An `<input type="text">` for the customer's name
- [ ] An `<input type="email">` for their email
- [ ] At least one set of `<input type="radio">` buttons (pick a meal size, crust type, etc.)
- [ ] At least one `<input type="checkbox">` (add toppings, sides, etc.)
- [ ] A `<select>` dropdown (pick a drink, delivery time, etc.)
- [ ] A `<textarea>` for special instructions
- [ ] A `<button type="submit">` to place the order
- [ ] Use `placeholder` on at least two inputs
- [ ] Use `required` on at least one input

### From Lesson 6 — Tables
- [ ] A `<table>` with `<thead>` and `<tbody>`
- [ ] At least 3 columns (Item, Description, Price)
- [ ] At least 4 rows of menu items

## Bonus Challenges

Done with the basics? Try adding these:

- **A second table** showing nutrition info or combo meal deals
- **A "Top 3 Sellers" ordered list** with `<strong>` on each item name
- **A "Catering" section** with a separate form that asks for party size (`<input type="number">`), event date (`<input type="date">`), and a message (`<textarea>`)
- **Multiple images** — one for each menu category
- **A navigation section** at the top with anchor links (`<a href="#menu">`, `<a href="#hours">`, etc.) that jump to each section using `id` attributes

---

## How to Test Your Page

1. Save your file as `index.html`
2. Find the file on your computer
3. Double-click it — it will open in your browser
4. Every time you make a change, save the file and refresh the browser

---

## Tips

- **Don't worry about making it pretty yet.** Styling comes in Lessons 8 and 9. Right now, focus on getting the right tags in the right places.
- **Use free images.** Go to [https://unsplash.com](https://unsplash.com) or use `https://picsum.photos/400/250` for a random placeholder photo.
- **Read your code out loud.** If something looks wrong in the browser, read through your HTML line by line. Usually it's a missing closing tag.
- **Have fun with it.** Make a taco truck, a fancy French bistro, a pizza shop on Mars — whatever makes you want to keep building.

---

## When You're Done

You should have a single HTML file that a real person could look at and understand what the restaurant serves, when it's open, and how to place an order. No CSS, no JavaScript — just clean, well-structured HTML.

You're ready for Lesson 7. 🎉
